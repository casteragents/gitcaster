# GitCaster Ref Consensus Public-Alpha Layer

GitCaster exposes the ref-consensus package as a public-alpha developer layer for local certificate work. It is meant for contributors who need to inspect how a local node accepts a signed ref update, issues a local ref certificate, appends that certificate to a local ledger, and records conflict or double-sign evidence.

This layer is not a public consensus launch. It does not prove public node quorum, remote ref durability, normal git transport, QStorage publication, CasterCloud deployment, or production runtime operations.

## Public Surfaces

- `packages/ref-consensus` contains certificate issuing, ref policy checks, append-only ledger helpers, certificate verification, conflict evidence, and double-sign detection.
- `apps/node/src/routes/refs.ts` shows the local node ref routes and keeps unsigned ref mutations blocked.
- `examples/refs/local-ref-certificate-workflow.example.json` is a redacted public fixture for certificate, conflict, and double-sign shapes.
- `launch/evidence/ref-consensus-local-certificate-source.json` is generated by the deterministic checker.

## Deterministic Checks

Run:

```bash
pnpm run ref-consensus:check
```

The checker proves:

- first local certificate issuance from `null`,
- second local certificate issuance from the current head,
- append-only local ledger verification,
- wrong `from` rejection with conflict evidence,
- branch deletion rejection,
- protected branch policy checks,
- double-sign detection,
- node-signature verification,
- forbidden live-status rejection,
- unsigned HTTP ref mutation blocking.

## Blocked Until Stronger Proof Exists

- Public consensus remains blocked until signed multi-node ref proof exists.
- Remote ref durability remains blocked until append-only remote ledger burn-in exists.
- Normal git transport remains blocked until pack transport, storage, and rollback proof exist.
- Production runtime remains blocked until managed ingress, deployment, audit, and rollback evidence exist.
